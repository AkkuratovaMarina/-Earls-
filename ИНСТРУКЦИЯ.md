# А теперь адекватная инструкция

## 1. Что нужно установить

- **Компилятор C++17**: например **MinGW-w64** (`g++` или `x86_64-w64-mingw32-g++`) на Windows, или `g++` / `clang++` на Linux/macOS.
- **GNU Make** или **`mingw32-make`** (часто идёт с MinGW/MSYS2).

Проверка в терминале:

---------------------------
g++ --version
make --version
---------------------------

Если системный `g++` не найден, но установлен кросс-компилятор с префиксом (как в примере ниже), его имя можно передать в `make`

## 2. Сборка

Откройте терминал в **корне проекта** (папка, где лежит `makefile`)

Сборка папки graphdro4:
### Windows (PowerShell, MinGW в PATH)

--------------------------------
mkdir graphdro4
copy *.hpp graphdro4\
--------------------------------

### Linux / macOS

--------------------------------
mkdir graphdro4
cp *.hpp graphdro4/
--------------------------------

Сборка самой программы:
### Windows (PowerShell, MinGW в PATH)

-------------------------
make clean
make all
-------------------------

Появятся `graphdro4.exe` и `tests_graphdro4.exe`.

### Linux / macOS

-------------------------
make clean
make all
-------------------------

Появятся файлы `./graphdro4` и `./tests_graphdro4`.

### Другой компилятор (имя отличается)

В `makefile` по умолчанию `CXX ?= g++`. Переопределите при вызове:

--------------------------------------------------------
make CXX=x86_64-w64-mingw32-g++ all
--------------------------------------------------------

или

--------------------------------------------------------
make CXX=clang++ all
--------------------------------------------------------

Флаги уже задаются в `makefile`: `-std=c++17 -O2 -Wall -Wextra -Iinclude -Ithird_party`.

### Только тестовый бинарник (без CLI)

------------------------------------------
mingw32-make tests_graphdro4
------------------------------------------

## 3. Смена кодировки(на всякий пожарный)

В терминале надо ввести:

----------------
chcp 65001
----------------

## 4. Запуск тестов (Catch2)

После успешной сборки:

---------------
make test
---------------

Это собирает `tests_graphdro4` (если нужно) и запускает его.

**Вручную:**

- Windows: `.\tests_graphdro4.exe`
- Linux/macOS: `./tests_graphdro4`

**Полезные опции Catch2:**

------------------------------------------------------------------------------------------------
tests_graphdro4.exe --list-tests          — список всех тестов
tests_graphdro4.exe "[parse]"             — только теги [parse]
tests_graphdro4.exe "[random]"            — тесты со случайностью / перестановками
tests_graphdro4.exe -s                    — успешные тесты тоже с подробным выводом
------------------------------------------------------------------------------------------------

При сбойном тесте Catch выведет **имя кейса** и **строку** в `tests_graphdro4.cpp`.

### Что именно проверяют тесты (кратко)

- базовые операции `Graph`, листья, смена бэкенда;
- парсеры: список рёбер, матрица, DIMACS, SNAP;
- DOT и `.edges`, кластеры EBCC/VBCC;
- метрики, мосты/точки сочленения, инвариант мостов при случайных перестановках вершин;
- **случайность**: остовное дерево на \(K_n\) при разных сидах даёт **разные** наборы рёбер; случайный простой цикл — **разные** циклы при переборе сидов; **G(n,p)** и **кубический** генератор — разные графы при разных сидах (типичный случай).

## 5. Консольная утилита `graphdro4`

Общий синтаксис: первый аргумент — **команда**. 

Вызов справки:

--------------------------
graphdro4 help
--------------------------

### 5.1. `backend-demo`

Демонстрация: один и тот же граф сначала в списке смежности, затем после `set_backend` — в матрице; число рёбер должно совпасть.

### 5.2. `parse`

------------------------------------------------------------
graphdro4 parse <format> <файл> [backend matrix|list]
------------------------------------------------------------

Форматы: `edge_list`, `matrix`, `dimacs`, `snap`.

Пример:

------------------------------------------------------------
graphdro4 parse edge_list data.txt
graphdro4 parse matrix graph.mat backend matrix
------------------------------------------------------------

### 5.3. `metrics`

-----------------------------------------------
graphdro4 metrics <format> <файл>
-----------------------------------------------

Печатает плотность, диаметр, транзитивность, компоненты, число артикуляций и мостов, двудольность, жадную оценку хроматического числа.

### 5.4. `gen` — генерация графа

-----------------------------------------------------------
graphdro4 gen <kind> <аргументы...> [выходной.dot]
-----------------------------------------------------------

Если последний аргумент — имя файла с **точкой** (например `out.dot`), граф дополнительно записывается в DOT с подсветкой случайного остова и цикла.

| kind | Аргументы |
|------|-----------|
| `kn` | n |
| `knm` | n1 n2 |
| `tree` | n |
| `star` | n |
| `cycle` | n |
| `path` | n |
| `wheel` | n |
| `gnp` | n p seed — модель Erdős–Rényi \(G(n,p)\), p от 0 до 1, seed — целое |
| `cubic` | n seed — случайный **простой** 3-регулярный граф (n чётное; при неудаче после попыток — вырожденный граф) |
| `components` | n k — k компонент размеров «почти равных», внутри компонент — клики |
| `bridges` | n b — конструкция с заданным числом мостов |
| `art` | n a — с артикуляциями |
| `ebcc` | n t — блоки с мостами между ними |
| `halin` | n [0\|1] [seed] — флаг «образца» и сид (см. `GenHalin` в коде) |

Примеры:

--------------------------------------------------------
graphdro4 gen kn 10
graphdro4 gen gnp 30 0.2 12345 picture.dot
graphdro4 gen halin 8 1 42
--------------------------------------------------------

### 6.5. `export`

-------------------------------------------------------------------------------------------
graphdro4 export dot <format> <файл> [out.dot] [cluster_e] [cluster_v] [tree] [cycle]
graphdro4 export edges <format> <файл> [out.edges]
graphdro4 export json <format> <файл> [out.json]
-------------------------------------------------------------------------------------------

`cluster_e` / `cluster_v` — кластеры рёберной / вершинной 2-связности в DOT; `tree` / `cycle` — подсветка случайного остова и цикла (сиды задаются в `DotSerializeOptions` в коде или через правку опций).
